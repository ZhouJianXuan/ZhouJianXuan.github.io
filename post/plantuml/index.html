PlantUML

PlantUML 简介

PlantUML 是一个用于生成各种图表的工具，它使用一种简单和直观的语言来定义图表，可以在很多其他工具中使用，可以生成 PNG，SVG 或 LaTeX 格式的图片，也可以生成 ASCII 艺术等。

PlantUML 原理

PlantUML 的原理是将用户编写的文本文件转换为图形文件。用户只需要按照 PlantUML 的语法规则来描述图表的元素和关系，然后 PlantUML 会根据这些信息来绘制图表。PlantUML 支持多种类型的图表，如序列图，用例图，类图，活动图，状态图等。

PlantUML 应用

PlantUML 的应用是广泛的，它可以帮助用户快速和方便地创建和更新图表，从而提高文档的可读性和可维护性。PlantUML 可以在很多其他工具中使用，如 VS Code，Eclipse，IntelliJ IDEA 等编辑器，或者 Markdown，LaTeX 等文档格式。PlantUML 也可以通过网页或命令行来使用。

PlantUML 优势

PlantUML 的优势是明显的，它相比于其他图表工具有以下几个优点：

  • 它使用文本文件来定义图表，这样可以方便地进行版本控制和协作。
  • 它使用一种简单和直观的语言来描述图表，这样可以减少学习成本和编写错误。
  • 它支持多种格式的输出，这样可以满足不同的需求和偏好。
  • 它支持多种类型的图表，这样可以覆盖不同的场景和领域。

示例

Example

@startuml
autonumber

actor "用户" as User
participant "登录Action" as Login
participant "登录界面" as LoginView
participant "UserService" as UserService
participant "登录首页" as LoginIndex

activate User
User -> Login: 访问登录界面
activate Login
deactivate User
Login -> LoginView: 转至登录界面
deactivate Login
activate LoginView
LoginView -> LoginView: 输入账号密码

activate Login
LoginView --> Login: 登录
deactivate LoginView

Login -> Login: 获取账号密码
Login -> UserService: 根据账号密码查询
activate UserService
UserService -> UserService: 查询User表
UserService --> Login: 返回User表
deactivate UserService

Login -> Login: 从返回表获取User
Login -> UserService: 根据id查找对应的角色
activate UserService
UserService -> UserService: 查询User表
UserService --> Login: 返回角色列表
deactivate UserService

Login -> Login: 将用户存入session
Login -> Login: 将信息写入日志文件
Login -> LoginIndex: 跳转到系统首页
activate LoginIndex
deactivate LoginIndex
deactivate Login
@enduml


@startuml

autonumber

actor "客户端" as User

participant "LocalServer" as LS #orange
participant "Server" as S

activate User
User -> LS: 开局，进行游戏进度推算，达到游戏结束条件
activate LS
LS -> User: 立即推送游戏结束，并将verify数据提供给客户端
deactivate LS

User -> S: 通过C2SLocalServerResultVerify传输verify数据
activate S
S -> S: 根据verify数据构建新的房间，并执行房间结算过程
S -> User: S2RGameOver,结算协议由服务端进行下发
deactivate S
deactivate User

@enduml
